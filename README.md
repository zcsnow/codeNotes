# code review 作用
   1、尽量避免bug的出现，降低bug的产出率，降低生产事故
   2、方便维护，降低维护成本，方便其他人接手
   3、正确处理业务需求逻辑，是否存在理解偏差，是否存在处理不当等
   4、尽量最优实现，在编码过程中，每个人的思维存在不同，评审时会发现更好，更优的方式或把最优的方式分享给其他人
   5、可以提高整体团队代码质量
   
   问题分类：代码问题+业务问题
   
   业务问题：业务处理、业务规范、规范习惯
   代码问题：（编码习惯、命名规范、日志打印、代码优化）、规范习惯、事物处理、异常处理、公共复用、bug、并发问题、代码注释
